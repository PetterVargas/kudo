## Descripción del Control

Controla y documenta la ubicación geográfica y lógica de todos los datos organizacionales, garantizando el cumplimiento de 
requisitos de residencia y soberanía de datos establecidos por regulaciones locales e internacionales. Incluye el mapeo de 
ubicaciones de almacenamiento (regiones de cloud, data centers físicos, sistemas de terceros), las restricciones de transferencia 
internacional de datos y los mecanismos de cumplimiento aplicables (cláusulas contractuales estándar, Privacy Shield, binding 
corporate rules), la evaluación de impacto para nuevas ubicaciones de datos y el monitoreo continuo para detectar transferencias 
no autorizadas de datos fuera de las jurisdicciones permitidas.

## Requisitos

## Requisito 1 - Mapa de Ubicaciones de Almacenamiento y Residencia de Datos

La organización documenta y mantiene actualizado el mapa de ubicaciones geográficas donde se almacenan y procesan sus datos (regiones 
cloud, data centers físicos, sistemas de terceros), identificando las restricciones de soberanía y residencia de datos aplicables por 
regulación para cada tipo de dato. La selección de regiones cloud para datos con requisitos de residencia es obligatoriamente validada 
por el equipo de seguridad y compliance. Se implementan controles técnicos (políticas de SCPs, Azure Policy) que impidan el almacenamiento 
de datos en regiones no autorizadas.

## Requisito 2 - Controles de Transferencia Internacional de Datos

Las transferencias de datos personales o confidenciales fuera de las jurisdicciones permitidas son evaluadas y autorizadas formalmente, 
documentando el mecanismo legal de cumplimiento aplicable (cláusulas contractuales tipo, normas corporativas vinculantes, decisión de adecuación). 
El monitoreo continuo detecta transferencias no autorizadas de datos hacia ubicaciones no permitidas. Las transferencias a terceros en el 
extranjero requieren un DPA actualizado que refleje los mecanismos de cumplimiento aplicables.
