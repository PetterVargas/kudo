## Descripción del Control

Mantiene un catálogo completo y actualizado de todos los datos que la organización recopila, procesa, almacena y transmite, 
clasificados según su sensibilidad y criticidad. Define los atributos mínimos del inventario (tipo de dato, clasificación, 
propietario, ubicación, sistema de origen, regulaciones aplicables), la taxonomía de clasificación de datos (público, interno, 
confidencial, restringido), el proceso de descubrimiento y clasificación automática de nuevos activos de datos, y las 
responsabilidades del propietario de datos (data owner) en cuanto a protección, acceso y retención. Constituye la base para todos 
los controles posteriores del dominio DIA.

## Requisitos

## Requisito 1 - Catálogo de Datos y Taxonomía de Clasificación

La organización mantiene un catálogo de datos completo y actualizado que registra todos los tipos de datos que recopila, procesa, 
almacena y transmite, con una taxonomía de clasificación de cuatro niveles (público, interno, confidencial, restringido). Para cada 
tipo de dato se documentan: propietario (data owner), custodio técnico, sistema de origen, ubicación de almacenamiento, regulaciones 
aplicables y tiempo de retención. El catálogo es actualizado ante nuevos proyectos y revisado integralmente al menos anualmente.

## Requisito 2 - Descubrimiento y Clasificación Automática de Datos

La organización implementa herramientas de descubrimiento y clasificación automática de datos que identifican y etiquetan datos 
sensibles (PII, datos financieros, datos de salud, secretos comerciales) en repositorios estructurados y no estructurados (bases de 
datos, file servers, object storage, correo electrónico, colaboración en la nube). Los datos clasificados automáticamente son 
validados periódicamente por los propietarios. Las etiquetas de clasificación se utilizan como base para la aplicación de controles 
de seguridad diferenciados.
