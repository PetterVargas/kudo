## Descripción del Control

Mantiene un registro completo y actualizado de todas las identidades —humanas (empleados, contratistas, proveedores) y 
no humanas (cuentas de servicio, service principals, API keys, managed identities, certificados de máquina)— que tienen 
acceso a recursos organizacionales. Define los atributos mínimos del inventario (ID único, tipo, estado, propietario, 
última actividad, fecha de revisión), los mecanismos de gestión del ciclo de vida (onboarding, cambios, offboarding) y 
la integración con el proveedor de identidad centralizado y el sistema de RRHH (HRIS).

## Requisitos

### Requisito 1 - Inventario de Identidades Humanas

La organización mantiene un registro completo y actualizado de todas las identidades humanas (empleados, contratistas, 
proveedores con acceso) que tienen acceso a recursos organizacionales, integrado con el sistema de RRHH (HRIS). El inventario 
registra para cada identidad: ID único, nombre, tipo (empleado/contratista/proveedor), estado (activo/inactivo), propietario, 
roles asignados, fecha de último acceso y fecha de revisión programada. El inventario es reconciliado automáticamente con 
el directorio activo/IdP al menos diariamente.

### Requisito 2 - Inventario de Identidades No Humanas

La organización mantiene un registro completo de todas las identidades no humanas: cuentas de servicio, service principals, 
API keys, managed identities, certificados de máquina y tokens de servicio. Cada identidad no humana tiene un propietario 
humano definido, un propósito documentado y una fecha de expiración. Las identidades no humanas sin propietario activo o sin 
uso en los últimos 90 días son identificadas automáticamente y desactivadas o eliminadas previo análisis de impacto.
