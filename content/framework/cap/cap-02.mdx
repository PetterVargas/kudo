## Descripción del Control

Implementa la integración continua de controles de seguridad en el pipeline CI/CD (DevSecOps). 
En CI: pre-commit hooks para detección de secretos y análisis de dependencias, SAST automatizado en 
el build, escaneo de imágenes de contenedor y análisis estático de IaC (Checkov, tfsec). En CD: DAST 
en ambientes de staging y validación de configuraciones desplegadas. Los security gates definen 
umbrales máximos de vulnerabilidades por severidad que bloquean automáticamente el despliegue. 
La gestión de secretos exige el uso de vaults e inyección en runtime, prohibiendo el almacenamiento 
de credenciales en código fuente o repositorios.

## Requisitos

### Requisito 1 - Pipeline CI/CD Seguro y Security Gates

El pipeline de integración y entrega continua (CI/CD) incorpora security gates automáticos que bloquean 
el progreso hacia producción cuando se detectan vulnerabilidades que superan los umbrales definidos. 
En la fase CI se ejecutan obligatoriamente: detección de secretos en código, SAST, análisis de dependencias 
(SCA) y escaneo de imágenes de contenedor. Los security gates definen umbrales máximos de vulnerabilidades 
por severidad (cero críticas/altas para producción). Los resultados quedan registrados en el sistema de 
tickets de seguridad.

### Requisito 2 - Análisis de Seguridad en Infraestructura como Código

Toda la infraestructura definida como código (IaC: Terraform, CloudFormation, Bicep, Pulumi) es sometida 
a análisis estático de seguridad (Checkov, tfsec, KICS) como parte del pipeline CI/CD, verificando el 
cumplimiento de las políticas de seguridad cloud (CIS Benchmarks, configuraciones seguras de CSP). Las 
desviaciones de seguridad en IaC bloquean automáticamente el despliegue. Las plantillas de IaC aprobadas sirven 
como referencia para configuraciones seguras.
