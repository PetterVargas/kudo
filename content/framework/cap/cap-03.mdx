## Descripción del Control

Asegura que todas las APIs —internas y externas— cumplan requisitos de seguridad a lo largo de su ciclo de vida. 
Requiere autenticación mediante OAuth 2.0/OpenID Connect o JWT firmados, con mTLS para APIs críticas y autorización 
granular (RBAC/ABAC). Exige validación de schema (OpenAPI/Swagger), sanitización de todos los inputs y protección 
contra OWASP API Security Top 10. Implementa rate limiting por usuario/IP/API key, throttling, circuit breakers y 
protección DDoS. El enmascaramiento de datos sensibles en logs y los headers de seguridad HTTP son obligatorios. 
Toda API debe estar inventariada, documentada y monitoreada.

## Requisitos

### Requisito 1 - Autenticación y Autorización en APIs

Todas las APIs de la organización (internas y externas) implementan mecanismos de autenticación robusta mediante 
OAuth 2.0, OpenID Connect o JWT firmados con algoritmos seguros (RS256, ES256). Las APIs críticas requieren mTLS 
para la autenticación mutua entre servicios. La autorización implementa control de acceso granular (RBAC/ABAC) 
siguiendo el principio de menor privilegio. Las API keys tienen fecha de expiración, son rotadas periódicamente 
y se revocan inmediatamente al finalizar su uso.

### Requisito 2 - Seguridad de Inputs y Protección OWASP API Top 10

Todas las APIs implementan validación estricta de esquema (OpenAPI/Swagger) y sanitización de todos los inputs 
para prevenir inyecciones (SQL, NoSQL, command injection, XXE). Los controles contemplan la protección contra 
las vulnerabilidades del OWASP API Security Top 10 incluyendo autenticación rota, autorización excesiva, exposición 
de datos sensibles, mass assignment y SSRF. Las pruebas de seguridad de APIs (DAST, fuzzing) son parte del pipeline CI/CD.

### Requisito 3 - Rate Limiting, Throttling e Inventario de APIs

Todas las APIs implementan rate limiting por usuario, IP y API key para prevenir abuso y ataques DDoS. Se implementan 
throttling y circuit breakers para garantizar la disponibilidad del servicio. El enmascaramiento de datos sensibles 
en logs de API y los headers de seguridad HTTP (HSTS, CSP, X-Frame-Options) son obligatorios. La organización 
mantiene un inventario completo y actualizado de todas sus APIs con datos de propietario, audiencia, versión vigente 
y estado de ciclo de vida.
