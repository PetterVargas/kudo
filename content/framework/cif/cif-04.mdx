## Descripción del Control

Establece los requisitos de cifrado para proteger la confidencialidad e integridad de datos en reposo y en tránsito en entornos 
cloud. En reposo exige cifrado en todos los servicios de storage (object, block, file, databases) con AES-256. En tránsito 
requiere TLS 1.2 como mínimo (preferible TLS 1.3) en todas las comunicaciones, con HTTPS obligatorio en APIs, mTLS para comunicaciones 
service-to-service y HSTS en aplicaciones web. Prohíbe explícitamente algoritmos débiles (DES, RC4, MD5, SHA-1). La clasificación 
de datos determina el nivel de cifrado requerido, siendo obligatorio para datos confidenciales y restringidos.

## Requisitos

### Requisito 1 - Cifrado de Datos en Reposo

Todos los servicios de almacenamiento en cloud (object storage, block storage, file storage, bases de datos relacionales y NoSQL) 
tienen habilitado el cifrado en reposo con AES-256 como mínimo. El cifrado es verificado automáticamente mediante políticas de 
cumplimiento del CSP. Datos clasificados como confidenciales o restringidos deben utilizar claves gestionadas por el cliente (CMK) 
almacenadas en KMS. Se prohíbe explícitamente el uso de algoritmos débiles o deprecados (DES, 3DES, RC4). El cumplimiento del cifrado 
es verificado en scans periódicos de configuración.

### Requisito 2 - Cifrado de Datos en Tránsito

Todas las comunicaciones de datos en tránsito entre sistemas, aplicaciones, APIs y servicios de la organización utilizan TLS 1.2 
como versión mínima, con preferencia por TLS 1.3. HTTPS es obligatorio en todas las APIs expuestas. Se implementa mTLS para comunicaciones 
service-to-service en arquitecturas de microservicios. Las aplicaciones web implementan HSTS con preload. Se prohíbe el uso de 
protocolos inseguros (HTTP, FTP, Telnet, SNMPv1). La configuración TLS es verificada periódicamente con herramientas de análisis SSL/TLS.
