## Descripción del control

Establece el sistema de gestión del ciclo de vida completo de claves criptográficas mediante servicios KMS gestionados 
(AWS KMS, Azure Key Vault, GCP KMS, HashiCorp Vault). Define una jerarquía de claves (Master Keys en HSM con rotación anual, 
Data Keys cifradas mediante envelope encryption con rotación trimestral, API keys con rotación mensual), controles de generación 
segura (CSPRNG), almacenamiento con separación por ambiente y propietario definido, distribución mediante canales seguros, 
auditoría de cada operación criptográfica y procedimiento de destrucción segura. La separación de funciones entre administrador y 
usuario de claves es obligatoria.

## Requisitos

### Requisito 1 - Jerarquía y Ciclo de Vida de Claves Criptográficas

La organización define e implementa una jerarquía de claves criptográficas mediante servicios KMS gestionados (AWS KMS, Azure Key 
Vault, GCP KMS) o HSMs para las claves maestras. La jerarquía incluye Master Keys con rotación anual almacenadas en HSM, Data Keys 
con rotación trimestral generadas mediante envelope encryption, y API keys con rotación mensual. Todas las operaciones criptográficas 
quedan registradas en logs de auditoría. La separación de funciones entre administrador y usuario de claves es obligatoria e 
implementada técnicamente.

### Requisito 2 - Generación, Almacenamiento y Destrucción Segura de Claves

La generación de claves criptográficas utiliza generadores de números aleatorios criptográficamente seguros (CSPRNG) y sigue los 
estándares FIPS 140-2/3 para los materiales criptográficos. El almacenamiento de claves está segregado por ambiente (producción, 
staging, desarrollo) con propietario definido para cada conjunto de claves. La distribución de claves se realiza mediante canales 
seguros y cifrados. La destrucción de claves al final de su vida útil sigue un procedimiento documentado que garantiza la eliminación 
segura y la verificación de la destrucción.
