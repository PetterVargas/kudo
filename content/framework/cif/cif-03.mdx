## Descripción del control

Implementa arquitectura de red segura en entornos cloud con segmentación por capas (DMZ, aplicación, datos, gestión) 
y principio default-deny en todos los controles de tráfico. Incluye uso de subnets privadas sin acceso directo a internet, 
WAF para aplicaciones web, protección DDoS, revisión trimestral de reglas de security groups, hardening de VMs basado en 
CIS Benchmarks, bastion hosts o servicios equivalentes como único punto de acceso administrativo con MFA y grabación de sesiones, 
network policies en Kubernetes, y habilitación de flow logs de VPC/VNet para monitoreo de tráfico anómalo.

## Requisitos

### Requisito 1 - Segmentación de Red y Arquitectura Zero Trust

La organización implementa segmentación de red en entornos cloud mediante subnets privadas diferenciadas por función (DMZ, 
aplicación, datos, gestión), aplicando el principio default-deny en todos los controles de tráfico (security groups, NACLs, 
network policies). Los recursos críticos no tienen acceso directo a internet. Se implementa WAF para la protección de aplicaciones 
web expuestas. La arquitectura sigue los principios de Zero Trust, verificando explícitamente la identidad y postura del 
dispositivo antes de otorgar acceso.

### Requisito 2 - Hardening de Sistemas y Acceso Administrativo Seguro

Todos los sistemas y máquinas virtuales en entornos cloud siguen una línea base de configuración segura basada en CIS Benchmarks 
para el sistema operativo correspondiente. El acceso administrativo remoto se realiza exclusivamente a través de bastion hosts 
o servicios equivalentes (AWS Session Manager, Azure Bastion) con MFA obligatorio y grabación de sesiones. Las reglas de security 
groups son revisadas trimestralmente para eliminar accesos no necesarios. Se habilitan los flow logs de VPC/VNet para monitoreo 
de tráfico anómalo.

### Requisito 3 - Protección DDoS y Monitoreo de Red

La organización implementa controles de protección contra ataques de denegación de servicio (DDoS) para todos los recursos 
expuestos a internet, utilizando servicios nativos del CSP (AWS Shield, Azure DDoS Protection) o soluciones dedicadas. Los flow 
logs de VPC/VNet están habilitados y centralizados en el SIEM para la detección de tráfico anómalo. Se definen umbrales de alerta 
para patrones de tráfico inusuales que disparan alertas automáticas al equipo de operaciones de seguridad.
